📖 Integration Guide
How to fetch data from Google Sheets
This dashboard reads live data directly from your Google Sheets using their public CSV export URL. No backend, no API keys needed.
Step-by-Step Setup
1
Publish your Google Sheet as CSV
Open your Google Sheet → File → Share → Publish to web → select your tab → choose CSV → click Publish. Do this for each tab/sheet separately.
2
Get your Sheet ID from the URL
Your sheet URL: https://docs.google.com/spreadsheets/d/YOUR_SHEET_ID/edit
Copy the long string between /d/ and /edit.
3
How the dashboard fetches data automatically
When you enter a Sheet ID and click Refresh, this URL is constructed and fetched:
// Auto-constructed URL — no code needed
const url = `https://docs.google.com/spreadsheets/d/${sheetId}/gviz/tq?tqx=out:csv&sheet=${tabName}`;
const csv = await fetch(url).then(r => r.text());
// Dashboard auto-parses CSV → JS objects → charts + tables
4
Enter your Sheet IDs in Data Sources
Go to Data Sources in the sidebar → paste the Sheet ID and tab name for each sheet → click Refresh All Data. The dashboard fetches and renders all data automatically.
5
Required column headers (Resources sheet)
The dashboard expects these exact column names (or remap in Employee Sheet → Configure Columns):
Code, Name, Email, Contract, Engagement, Title, BillingResourceLevel,
Competency, StartDate, EndDate, BillingType, AllocationPercentage,
ShiftAllowance, TrackMonths
// TrackMonths = add as a formula: months since track start date
Troubleshooting
!
Fetch blocked / CORS error?
Ensure the sheet is Published to web (not just shared). Org-restricted sheets won't work. Set sharing to "Anyone with link → Viewer" AND publish via File → Publish to web.
?
Wrong tab name?
Tab names are case-sensitive. Check the tab label at the bottom of your Google Sheet. Resources ≠ resources.
✓
Data not updating?
Google caches published CSVs for ~5 minutes. After editing your sheet, wait a few minutes then click Refresh All Data in the sidebar.
